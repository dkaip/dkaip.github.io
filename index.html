Java bindings to the Vulkan® graphics API

This project was created to provide Java programs access to the Vulkan® SDK available from LunarG. This access is provided in a much more Java friendly manner than other available options.

Java Bindings

The following two projects:
jvulkan https:://github.com/dkaip/jvulkan The Java version of the Vulkan® API.
jvulkan-natives-Linux-x86_64 https:://github.com/dkaip/jvulkan-natives-Linux-x86_64 A native library required to use jvulkan.
provide a way for Java programmers to create applications that use the Vulkan® drivers available for their graphics card(s).

Example Applications

This project contains a number of working examples that use the jvulkan library mentioned above.
jvulkan-examples https:://github.com/dkaip/jvulkan-examples Examples of applications that use jvulkan

Java 3D

This project is the Java3D that was available a long time ago. I have worked on it to bring it to a more modern state. Things like creating Java enums instead of using ints for all of the data items that should have been enums in the first place, but, at the time Java did not have an enum type. In addition the inner working have been somewhat unscrambled to make it much easier to follow what is going on in the code. It just runs multithreaded now instead of having the option not to. The last thing I am working on / will work on is to make it able to use Vulkan® as its interface to a graphics card.

Java Wayland Interface

This project jwayland was created so that Wayland could be easily used in a Java application. Mainly in order to get access to the events directly in a Java application. It is partially working at this point. I will need to finish it and probably have to create a Java interface to EGL as well in order to be truely useful.

SECS Tools

This section contains references to projects that have relevence in the SEMI SECS domain. If the words SEMI and SECS together don't mean anything to you these projects are probably not going to be of any interest to you. SEMI refers to "http://www1.semi.org" and SECS is short for "SEMI Equipment Communications Standard".

Station Controller

A program that can be used in the development and debugging of equipment interfaces in the semiconductor shop floor environment. It may also be used to construct a full flegded station controller if your application is not very complicated. By complicated I am thinking of a 300mm plus environment with AMHS, RTD, etc.
sc_open https:://github.com/dkaip/sc_open Station Controller

JavaSECSTools

JavaSECSTools https:://github.com/dkaip/JavaSECSTools A Java library with classes and methods to provide ways to encode, decode, and manipulate SECS II messages.

CSharpSECSTools

CSharpSECSTools https:://github.com/dkaip/CSharpSECSTools A CSharp library with classes and methods to provide ways to encode, decode, and manipulate SECS II messages.

Java JNI

Java JNI can be a real pain if you are just beginning with it. If you would like to do more than just trivial stuff with JNI take a look at Java Bindings in the Java bindings to the Vulkan® graphics API project. In the jvulkan and the jvulkan-natives-Linux-x86_64 repositories you will find examples of using JNI to work with enum(s), EnumSets, Objects, Collections of objects, etc. that are mostly well beyond what I have been able to find on the web. I am not saying that my way accessing Java objects from a c++ environment is the best way, but, it works.
